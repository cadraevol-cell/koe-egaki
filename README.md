# こえ描き（VoiceDoodle）

声の高さでペンを上下させ、右へ進み続けて右端に着くと左端から現れるペンで絵を描くブラウザお絵かきゲームです。
上に出る「絵のお題」を見ながら、右下のボタンを長押ししたタイミングだけ色が塗られる、少しシュールな創作ゲーム。
描き終わったら「完成」ボタンで結果画面に切り替わり、審査員パンダ（ちょっと皮肉屋）がお題にどれくらい近いかを
ジャッジします。結果画面からは「次のお題へ」でもう1枚、「タイトルへ戻る」でホームに戻れます。

完全にクライアントサイドのみで動作します（サーバー・DB・外部API・課金なし）。`index.html` 1ファイルに
HTML / CSS / JavaScript がすべて内包されています。

## あそびかた

1. ホーム画面で「はじめる」を押してマイクを許可する。
2. ゲーム画面が開くと、上部に「絵のお題」がランダムで表示される。この時点からすでにペンは縦方向（声の高さ）に
   反応する。まだ横には動かないので、スタート前に声で高さを合わせておける。
3. 右下の「スタート」ボタンを押すと、ペンが左端から右へ進み始める（右端に着いたら左端から再び現れる）。
4. 右下のボタンが「描く」に変わるので、長押ししている間だけ色が塗られる。声を出してペンの高さを操作しながら、
   良いタイミングで長押しして絵を描こう。
5. 右のパレットで色を選べる。一番下の虹色の丸をタップすると、好きなRGBカラーを自由に選べる（無限の色）。
6. 上部のボタンから「元に戻る（直前のひと塗り/クリアを取り消し）」「クリア」「保存(PNG)」「シェア」
   「お題チェンジ」「一時停止」「ズーム」「なぞーる」「ホームに戻る」を操作できる。「一時停止」を押すと
   ペンの動き（横方向も、声に反応する縦方向も）が完全に止まり、もう一度押すと再開する。
   「🔍 ズーム」はキャンバス上で**マウスホイールを回す**と連続的にズームイン/アウトできる（PC向け）。
   ボタンをタップすると 1x ⇔ 2.5x をトグルする（ホイールが使えないスマホ向け）。ズームしてもペンの
   実際の移動速度・見た目のサイズは変わらないので、拡大するほど相対的に細かい部分を狙って描きやすくなる。
   「📝 なぞーる」をONにすると、キャンバスに現在のお題を表す絵文字が薄いグレーの下絵として表示され、
   それをなぞるように描ける（下絵は保存・シェアした画像には写り込まない）。
7. 描画エリアの下にある3つのスライダー（ペンの速さ・ペンの太さ・声の感度（音域幅））は常時表示されており、
   いつでも調整できる。
8. 描き終わったら右下の緑の「🏁 完成」ボタンを押すと、**結果画面に切り替わって**判定結果と、実際に描いた絵の
   プレビュー（右側）が表示される。結果画面には「↩ 戻って続ける（今の絵のまま描画に戻る）」「保存(PNG)」
   「タイトルへ戻る」「次のお題へ」のボタンがあり、次のお題へ進むと新しいお題でキャンバスがまっさらな状態
   から再スタートできる。

- 高い声 → ペンが上へ
- 低い声 → ペンが下へ
- 声を出していない間はペンのY位置はそのまま保持される
- キャンバスは正方形。その左右には半透明の空・山のミニ背景エリアがあり、ここは塗れないがペンは通過し続け、
  声の高さもここで反映される（キャンバスに入る前に音程を合わせておける練習ゾーン）
- 保存した画像は正方形の描画部分のみが書き出される（左右のエリアは含まれない）

### 完成度判定について

「完成」ボタンを押すと結果画面に切り替わり、キャンバス上のインクの分布から

- お題ごとに設定した「典型的な縦横比（幅/高さ）」との近さ
- お題ごとに設定した「典型的な塗りの密度」との近さ
- 塗った量（コピー量）や使った色数

を測定し、0〜100%のスコアと審査員パンダの皮肉気味なコメントを表示します。これは**本格的な画像認識AIではなく**、
形状の統計的な特徴（バウンディングボックスの比率や塗りの密度）とお題ごとの想定プロファイルを突き合わせる
ヒューリスティックです（完全無料・オフライン動作の制約上、実際に「猫に見えるかどうか」を認識するような
本物の画像認識モデルは使っていません）。

## ローカルでの起動方法

マイク（`getUserMedia`）は **HTTPS** または **localhost** でしか使えません。ファイルを直接ダブルクリックして
`file://` で開くと動作しないため、必ずローカルサーバー経由で開いてください。

```bash
# このフォルダ（voice-doodle）に移動してから
python3 -m http.server 8000
```

ブラウザで [http://localhost:8000](http://localhost:8000) を開いてください。

Python が無い場合は、Node.js の `npx serve` や VS Code の「Live Server」拡張などでも代用できます。

```bash
npx serve .
```

## スマホでの利用について

- iOS Safari / Android Chrome の両方で動作します。
- iOS ではマイク許可のダイアログが出ます。許可してください。
- iOS の `AudioContext` はユーザー操作なしでは自動再生されないため、「はじめる」ボタン（ユーザー操作）をトリガーに
  `AudioContext.resume()` を呼ぶ実装になっています。
- スマホではスマホ本体の「サイレントモード」やマイク権限設定によりマイクが使えない場合があります。その場合は
  ブラウザ・OS 設定でマイクへのアクセスを許可してください。

## 無料デプロイ手順（GitHub Pages）

1. GitHub で新規リポジトリを作成し、`index.html`（と `README.md`）を push する。

   ```bash
   git init
   git add index.html README.md
   git commit -m "Add こえ描き"
   git branch -M main
   git remote add origin https://github.com/<ユーザー名>/<リポジトリ名>.git
   git push -u origin main
   ```

2. リポジトリの `Settings` → `Pages` を開き、`Branch` を `main` / `root` に設定して保存する。
3. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` が発行されるのでアクセスして動作確認する。

GitHub Pages は HTTPS で配信されるため、マイクもそのまま利用できます。

## 実装メモ

- ピッチ検出は外部ライブラリを使わず、`AnalyserNode.getFloatTimeDomainData` で取得した波形に対する自己相関法で
  自前実装しています（無音判定・人声域70〜1000Hzへのクランプ・中央値+移動平均によるスムージング・急激な変化の
  上限クランプ付き）。マイク入力ノード（`MediaStreamAudioSourceNode`）は `state.micSource` として明示的に
  保持しています。これは、出力先（destination）まで繋がっていないノードをブラウザがガベージコレクションして
  しまい、しばらくすると無音扱いになる既知の挙動を避けるためです。
- 縦位置へのマッピングは対数スケール（`log2(pitch)`）で行っており、設定パネルの「声の感度（音域幅）」スライダーで
  調整できます。
- ペンは横方向に一定速度で進み続け、右端に到達すると左端にワープして進み続けます（往復（ピンポン）ではなく
  一方向ループ）。ペンのカーソルは丸ではなく、実際のペン先が描画点に来る向きで描画したアイコンです。
- 描画は Canvas 2D API のみを使用しています。描画用キャンバス（正方形）とペンカーソル用のオーバーレイキャンバス
  （左右のミニ背景エリアを含む全体の幅）を重ねており、カーソルが保存画像に写り込まないようにしています。
  ペンは常にオーバーレイキャンバス上の座標で移動し、実際に塗る（`lineTo`）操作をする際だけ正方形キャンバスの
  ローカル座標に変換しています。
- アニメーションループはゲーム画面に入った瞬間から常時動いており、スタート前（ready状態）は縦方向（声の高さ）
  だけを反映し、横方向は動かしません。「一時停止」ボタンを押すとこのループ全体（縦横どちらの動きも）が
  完全に止まり、もう一度押すと再開します。結果画面（完成画面）に切り替わるときもループを一時停止し、
  「次のお題へ」を押すと新しいお題・空のキャンバス・「ready」状態（スタート待ち、縦だけ反応）で再開します。
- 左右のミニ背景（空・太陽・雲・山）は画像を使わず、CSSのグラデーションと`clip-path`だけで描いています。
- ズームは「レイヤー1（キャンバス＋左右のミニ背景、`.camera`）」と「レイヤー2（ペン、`#cursorCanvas`）」を
  分離することで実現しています。**カメラはペンを追従しません** -- `.camera`は常にプレイエリア全体の中心を
  基準に拡大するだけの固定変形（`transform: scale() translate()`、中心点は`state.zoom`が変わるとき以外は
  動かない）です。その代わり、ペンの横方向の移動（`state.penX`）はキャンバス座標ではなく**画面（ビューポート）
  座標**で管理しており、ズームの有無にかかわらず「画面の端から端まで常に同じ実時間・同じ見た目の速度」で
  往復します。実際にインクを置く位置を決めるときだけ、`screenXToRealX()`でその画面座標をズーム倍率に応じて
  実座標へ逆変換します（ズームするほど、同じ画面移動量に対応する実キャンバス上の移動量が小さくなる＝細かく
  描けるようになる、という仕組みです）。縦方向は逆に声の高さが実キャンバス座標を直接動かすので、ペン
  （レイヤー2）を画面に描画する際だけ`realYToScreenY()`で順方向に変換しています。ペン自体は`.camera`の外側の
  兄弟要素なので拡大縮小の影響を受けず、常に同じ見た目のサイズで表示されます。ズーム値は1〜4倍の連続値
  （`state.zoom`）で、`play-wrap`上の`wheel`イベントで`deltaY`を指数関数的に速度へ変換して滑らかに
  変化させています。
- 結果画面は「戻って続ける」で今の絵のまま描画に復帰できます。完成ボタンを押した時点では`state.gameState`を
  変更していない（`"playing"`のまま）ため、結果画面から戻る際はアニメーションループを再開するだけで、
  キャンバスの内容・お題・ペン位置はすべて中断前と同じ状態から続けられます。プレビュー画像は保存/シェアと
  同じ`buildSaveCanvas()`（白背景合成＋任意でお題の透かし）の出力をそのまま`<img>`に表示しています。
- 「元に戻る」は、ひと塗り開始前やクリア実行前のキャンバス状態を `ImageData` としてスタックに保存し、
  ポップして復元する簡易Undoです（最大12件）。
- 「なぞーる」の下絵はお題ごとに割り当てた絵文字（`TOPIC_EMOJI`）を、Canvas 2Dの`filter`プロパティ
  （`grayscale(100%) opacity(20%)`）を使って薄いグレーで`#guideCanvas`という別レイヤーに描画しています。
  描画用キャンバス（`#drawCanvas`）自体は透明にしてあり（白い紙の見た目は`.canvas-wrap`のCSS背景色による
  もの）、その透明なキャンバスの下に下絵レイヤーを、上にペン用オーバーレイを重ねる3層構成にすることで、
  実際に塗った部分だけが下絵の上に乗り、保存・シェア時は`#drawCanvas`だけを書き出すので下絵は含まれません。
  この変更に伴い、完成度判定のインク検出も「白に近い」だけでなく「透明（アルファ値が低い）」も未塗装として
  扱うよう対応しています。
- 色は固定10色のプリセットに加えて、`<input type="color">` のネイティブカラーピッカーでRGBを自由に指定できます。
- 保存画像の右下に、任意で「絵のお題」を小さく焼き込んでいます（`state.showTopicOnSave` で切り替え可能）。
- 「判定」機能はキャンバスのインクのバウンディングボックス比率・塗り密度・塗った量・使用色数を、お題ごとに
  定義した `TOPIC_PROFILES` の期待値と比較するヒューリスティックです。外部APIやMLモデルを使わない制約のため、
  本物の画像認識ではなく形状の統計的特徴による近似判定である点に注意してください。
- 絵のお題（`PICTURE_TOPICS`）は角ばった・平面的なシルエット（山、家、ケーキ、帽子など）中心に絞っています。
  声だけで操作するこのペンは滑らかな円弧を描くのが苦手で、丸系のお題（太陽・りんご・風船など）は逆に難しく
  なりがちなため、多少ガタついても「それっぽく」見えやすい形だけを残しました。
